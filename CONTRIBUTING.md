# Contributing

Thanks for improving Code-Testing-Skill-Claude-Codex!

## Development Flow

1. Fork or branch from the default branch.
2. Install dependencies using the commands in `README.md`.
3. Make a focused change with tests or validation notes.
4. Run the available checks before opening a pull request.
5. Keep PRs small enough to review.

## Pull Request Checklist

- [ ] Documentation updated when behavior changes
- [ ] Tests/checks run locally, or reason noted
- [ ] No secrets, generated databases, or private files committed
- [ ] Screenshots/GIFs updated for UI-facing changes
